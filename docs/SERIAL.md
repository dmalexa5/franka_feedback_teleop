# Serial Device Access

This container can access `/dev/ttyACM0` as the non-root `user` by passing the device through and adding the host device's numeric group ID at runtime.

## Docker Run

Build the image, then start it with the device and its host GID:

```bash
docker run --rm -it \
  --device=/dev/ttyACM0:/dev/ttyACM0 \
  --group-add "$(stat -c '%g' /dev/ttyACM0)" \
  <image>
```

## Docker Compose

Export the device group ID before starting Compose if `.env` was not auto-generated by the devcontainer init step:

```bash
export SERIAL_GID="$(stat -c '%g' /dev/ttyACM0)"
docker compose up --build
```

The devcontainer initialization command writes `SERIAL_GID` into `.env` automatically when `/dev/ttyACM0` exists on the host.

## Validation Inside The Container

Confirm the device is present:

```bash
ls -l /dev/ttyACM0
```

Confirm the process has the matching supplemental group:

```bash
id
```

Confirm read and write access from Python:

```bash
python3 -c "import os; print(os.access('/dev/ttyACM0', os.R_OK | os.W_OK))"
```
